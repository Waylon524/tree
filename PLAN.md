# Project Plan

此文件只保留已经讨论并确认、但尚未完成的项目计划。它不是开发日志，也不记录已完成事项；历史变更请查看 [CHANGELOG.md](CHANGELOG.md)。

## 已确认的待办

### NodeRun 快速模式

目标是在保留现有标准模式的同时，为追求生成速度和调用成本的用户提供只调用一次 Writer 的快速模式。设置页增加 `标准模式 / 快速模式` 切换，默认保持标准模式。

#### 运行语义

- 标准模式继续执行 Examiner → Student → Examiner → Writer 的命题、作答、审核、修订循环，现有行为不得回退。
- 快速模式每个知识节点只调用 Fast Writer，不生成试卷、答案或学生作答，不执行 Examiner、Student、reconciliation、重复 Bottleneck 和迭代质量审核；provider 传输重试、请求终态检查仍保留。
- 模式在节点真正启动时确定并持久化到对应 NodeRun。运行中切换时，所有已经启动的并发节点保持各自原模式，之后启动的节点使用新模式。
- 普通暂停、失败恢复继续使用该节点已保存的模式和检查点；“重新生长”清除模式快照，使节点按重新启动时的当前设置选择模式。切换模式不自动重写已经完成的知识文件。

#### Fast Writer 输入与职责

- 快速模式使用当前 KnowledgeNode 及其 NodeRun 可见前置节点的 RAG 资料；当前节点必须覆盖 Dagger 已确定的 `member_mtu_ids` 和 `defines`，前置节点只作为已经掌握的衔接与引用范围。
- Dagger 仍独占 MTU 归组、KnowledgeNode 边界、`member_mtu_ids`、`defines` 和先修关系的决定权。Fast Writer 不得重新分组、拆分、遗漏或改派 MTU，只负责把已归入当前节点的证据按教学逻辑综合成一篇连贯教材，不能按 MTU、RAG 分块、来源文件或例题编号机械拼接。
- 允许直接引用或改写原始资料，但 RAG 只是参考证据而非绝对正确的权威。Writer 必须检查事实、术语、公式、推导和内部一致性；原文错误、不完整或互相矛盾时应输出正确内容，不得照抄错误。
- 输出继续经过现有确定性标题、先修前置、学习目标、来源追溯、原子发布、Ledger 和 finished RAG 索引流程，最终知识文件格式必须与标准模式一致。

#### 快速模式提示词

- 增加独立的 Fast Writer system prompt，但以现有 Writer prompt 为基线，继承代码控制/不可信材料分区、Dagger 固定成员 MTU 与 defines、材料外基础最小解释桥、证据限定引用和程序管理区块等当前契约，并尽可能保持教材风格、ActiveNode 增量边界、零基础教学、定义与公式说明、完整推导、例题、误区、Markdown、LaTeX 和固定章节结构不变。
- 只删除或替换多智能体协作内容：Bottleneck 驱动的 CREATE/OPTIMIZE、缺陷修补、`VALIDATED_WRITER_INSTRUCTIONS_JSON`、Exam Confidentiality、Examiner 命名范围、`prerequisite_repairs`、试卷/答案和审核结果检查。
- 用代码生成且不可由 RAG 覆盖的 `FAST_WRITER_TASK_SPEC_JSON` 代替 Examiner Writer Instructions，至少声明节点 ID、标题、defines、直接前置、可见祖先、禁止扩展的兄弟/未来节点和 `fast_create` 模式。
- 将原 prompt 中“整合多个 source chunks”改写为明确的职责边界：节点和 MTU 归属已经由 Dagger 固定，Writer 仅综合已提供的成员 MTU 证据并覆盖节点 defines，不进行再次聚类或节点拆分。

#### 快速模式门禁

- 快速模式没有试卷、答案或学生作答输入，因此不执行考试内容清洗，也不使用“试卷/标准答案标题”输出拒绝规则，避免误删课件中合法的习题答案和解析；标准模式的现有考试保密门禁保持不变。
- RAG、原文和前置资料仍必须作为不可信数据封装，禁止其中的提示词或控制语句覆盖 system prompt 和代码声明的任务边界。
- 不增加语义质量审核，但保留非空教学正文、节点范围、固定文件结构、来源完整性、事务一致性和索引一致性等确定性完整性检查。这些门禁不得触发三智能体循环或额外内容修订调用。

#### 验收标准

- 设置 API、桌面设置页和中英文文案能够读取、保存并显示标准/快速模式。
- 快速节点的 operation 记录中只有 Fast Writer 内容调用，没有 Examiner、Student 或 reconciliation 调用；标准节点调用序列保持不变。
- 快速模式产物与标准模式具有相同的 H1、先修前置、六段教学结构和来源追溯，且正常进入 Ledger 与 finished RAG。
- 覆盖运行中双向切换、多个并发已启动节点、暂停续跑、失败重试、重新生长和已完成节点不重写等状态测试。
- 覆盖多 MTU 节点综合、前置 RAG 边界、合法“标准答案/解析”资料不被误删、空输出拒绝、提示词注入隔离以及标准模式无回归测试。

## 更新方式

- 只保留当前有效的目标、范围、关键约束和验收标准。
- 计划完成、取消或被替代后，及时移除或改写对应内容，不保留完成任务清单。
