T.R.E.E. 项目完整工作流程

资料驱动、planner 选定完整知识节点、writer 完整写作、examiner 以考促写验收、PASS 后入库并刷新图谱。

资料摄入与 RAG
候选节点与 Planner
以考促写循环
入库与续跑
1. 资料摄入

用户放入原始资料

PDF、图片、DOCX、Markdown、TXT、PPT 等进入 materials / raw_materials。

↓

OCR / 文档提取

PaddleOCR 或文件提取器读取文本、版面、公式、列表与表格。

↓

Archivist 结构化

清洗 OCR 噪声，修正结构，重组成教材可用 Markdown。

  • 保留实质内容
  • 规范标题与公式
  • 写入 source_materials/<collection>
↓

Source RAG 入库

source Markdown 被 chunk、embedding 后写入 Qdrant；embedding 成功后删除中间 Markdown。

2. Planner 建图

构建 Source Inventory

从 source RAG chunk 提取概念、先修、方法、公式、summary。

↓

生成 Candidate Nodes

按概念、先修、方法、signature、相邻 section 聚类，并可由 AI enrichment 增强。

  • 节点 ID 基于 chunk refs 稳定哈希
  • 保留 estimated_output_lines / size_fit / size_band
↓

构建 Knowledge Graph

结合候选节点与 finished ledger，计算覆盖、重复、合并、先修与相邻关系。

↓

Planner 选择 selected_node

根据 root_score、support_score、size_fit、evidence_strength 选择下一个完整写作节点。

3. 以考促写

Step 1 Examiner 命题

在 planner selected_node 范围内生成知识点、盲考试卷、标准答案与 writer 指令。

↓

Step 2 Student 盲测

student 只能读取 prior finished outputs 和当前 draft，不读取 source RAG。

↓

Step 3 Examiner 审计

同时检查答案正确性、草稿支撑性、知识缺陷与越界使用。

↓

PASS 还是 FAIL？

PASS 入库；FAIL 进入 writer 修复，同一试卷继续重测。

4. 写作与入库

Step 4 Writer 写 / 修 draft

writer 必须完整写 planner-selected node，不因长度或过宽拒写。

  • 不返回 EXAM_TOO_BROAD
  • 不运行拆分反馈
  • 只写 selected node delta
↘

drafts 进入重测

草稿写入 drafts/<chapter> 后回到 Student 盲测，直到 examiner PASS。

↓

PASS 后入库

draft 移动到 outputs / finished_outputs，并索引为 finished RAG。

↓

Ledger + Graph 刷新

记录 graph_node_id、hit_chunks、covered_concepts；刷新 graph 后 planner 继续选下一个节点。
